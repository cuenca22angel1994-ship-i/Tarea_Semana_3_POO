Descripción de los programas desarrollados
 
1. Programación Tradicional
 
Se creó un sistema sencillo en el archivo tradicional.py, ubicado en la carpeta Programacion_tradicional. Funciona mediante variables y funciones, sin usar clases ni estructuras complejas. Permite ingresar por teclado el nombre, especie y edad de una mascota, valida que los datos sean correctos y muestra la información en forma ordenada. Todo se organiza en procedimientos independientes que operan sobre los datos.
 
2. Programación Orientada a Objetos
 
Se estructuró el sistema en dos archivos dentro de la carpeta Programacion_poo :
 
* mascota.py: Define la clase  Mascota , que agrupa los atributos (nombre, especie, edad) y los métodos para mostrar información y simular el sonido que emite. Aplica el principio de abstracción al ocultar los detalles internos y exponer solo sus funcionalidades.
* main.py: Crea instancias (objetos) de la clase y ejecuta sus métodos para mostrar los datos de cada mascota.
 
(Reflexión sobre las diferencias)
 
La programación tradicional es práctica para tareas simples, mientras que la orientada a objetos ofrece mayor orden, flexibilidad y facilidad de mantenimiento, especialmente en sistemas que pueden crecer con el tiempo.
